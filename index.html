About this sync: Your client data is saved as a single file in your Google Drive (look for pallet_portal_data.json). Changes are synced automatically after each save. If you edit on another device, click "Pull from Cloud" to fetch the latest version before making changes.

To sync across devices, you'll create free Google API credentials (~15 minutes, one time). Your data stays in your own Google Drive — nothing goes to anyone else.

Step 1 — Create Google Cloud Project
Go to console.cloud.google.com, sign in, click "New Project", name it (e.g. "Pallet Portal"), and create it.
Step 2 — Enable Google Drive API
In your project, go to APIs & Services → Library, search "Google Drive API", and click Enable.
Step 3 — Configure OAuth Consent Screen
Go to APIs & Services → OAuth consent screen. Choose "External", fill in app name (e.g. "Pallet Portal"), your email, and save. Under "Scopes" add drive.file. Under "Test users" add your own Google account.
Step 4 — Create OAuth Client ID
Go to Credentials → Create Credentials → OAuth client ID. Type: "Web application". Under "Authorized JavaScript origins" add the URL where you'll host this file (e.g. https://yourname.github.io). Copy the Client ID.
Step 5 — Create API Key
In Credentials, click Create Credentials → API Key. Copy it. (Optional: restrict to Drive API for security.)
Step 6 — Paste credentials below
Both are stored locally on this device only.
Compliance note: Storing real client financial data on personal Google Drive may not comply with Thailand's PDPA or AIA's data policies. Verify with AIA compliance before using with real client data. For sample/demo data or your own planning, this is fine.
